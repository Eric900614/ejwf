# 驾驶舱是本地 app，不是托管网页

## 决定

驾驶舱作为一个**跑在用户自己机器上的本地应用**交付，而非托管在服务器上的网页。它从 GitHub API 读卡片/标签/PR，从**本机文件系统**读 repo 内文档与代码（`docs/adr/`，及后续 Pillar C 的源码）。

## 背景 / 为什么

两件事只有「住在用户机器上的程序」才做得到，而它们是产品的命脉：

1. **读本地 repo 文件** —— 「为什么链」要读 ADR（`docs/adr/NNNN-*.md`），架构图（Pillar C）要读源码。托管网页够不着用户磁盘。
2. **拉起本地 agent** —— Phase 2/3 的灵魂是触发 CC/Codex 干活，而这些是用户**本机上的进程**；托管网页碰不到用户的机器，这条路从根上不通。

加上本产品是**自用、单人**（见 ADR 思路与 CONTEXT），无需为多用户/登录/托管密钥付代价；GitHub token 与 API key 留在本机更省心。

## 后果

- 放弃「随处可开（手机/别的电脑）」。对自用工具可接受。
- 需要本机常驻运行。
- 读 ADR / 代码倾向于读 GitHub `main` 分支的版本，而非本地可能脏、可能在别的分支、可能有多个 worktree 的副本，以避免「读哪一份」的歧义（细节待定）。
