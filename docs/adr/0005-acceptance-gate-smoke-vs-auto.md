# 卡片验收闸门：smoke vs 自动放行，及 `needs-smoke` 信号

## 决定

一张 **卡片** 在 **开发–审查环** 走完（代码合入）之后，怎么 **验收**（判「真做完了、可放行到 完成」）：

1. **判据 = 这张卡的"做完"机器判不判得了**：
   - **机器可判**（纯逻辑卡 + 行为测试齐全 + CI 绿）→ **自动放行**（**自动段**），不惊动维护者。
   - **机器判不了**（要肉眼看 UI/体验、要端到端联调"感觉对不对"）→ 停在 **smoke 人工闸门**，等维护者亲自跑起来拍板。
2. **smoke-or-not 在 triage 期决定**——拆票/triage 时已知这一刀是 UI 还是纯逻辑，故把它当 triage 的一项产出，用一个 **`needs-smoke` 标签**显式承载在卡片上。
3. 这是对 **ADR-0001「不新增标签、不改写卡习惯、一律从现有信号派生」的一次有意识、有限的破例**：因为「要不要 smoke」确实**派生不出来**，且它直接服务产品北极星（压低 **HITL 占比**）。
4. 对"机器可判、可自动放行"的卡，**终点是全自动关卡**（工具自己关卡、维护者不照面，HITL 最低）。但**分阶段抵达**：先经「**自动归堆 + 维护者批量瞄一眼批准**」的过渡档攒信任、盯工具会不会误判，信得过后再撤掉那道眼、滑到全自动。且**按卡分速**——纯逻辑 + 测试厚的卡尽快放手到全自动，混合卡 / 判定弱的卡在过渡档多留一会儿人。

## 背景 / 为什么

产品北极星是**尽可能释放 Agent 产能、压低维护者的 HITL 占比**（见 `CONTEXT.md` 的「人在环」「人工闸门是会移动的前线」）。要把维护者从「做完之后逐张盖章」里解放出来，工具就必须**自动认得出哪些卡能自动放行**。

而「这张卡要不要 smoke」**无法从标签/正文可靠派生**——它取决于卡的性质（界面 vs 纯逻辑），机器读正文猜不准。但这个判断在 **triage 期天然已知**（维护者定栈、拆票、写 brief 时就清楚这一刀要不要肉眼验收）。于是最顺的落点是：triage 期决定、盖一个标签。

本判据并非凭空新增，而是**复用已定的两条原则**：「前端体验是人工闸门」（`CONTEXT.md`）、「纯逻辑模块写'输入→断言输出'行为测试」（PRD #1）。`needs-smoke` 只是把"这张卡落在这两条原则的哪一边"显式记下来。

## 被否决的选项

- **启发式猜**（看卡动了哪个模块/带什么标签来推断要不要 smoke）——零新约定，但**会猜错**：要么漏掉该 smoke 的卡（坏东西被自动放行），要么瞎拦本可自动的卡（白白留住维护者）。验收是"敢不敢放行"的关口，猜错代价高。
- **事后人肉当场判**（不预分类，卡片 PR 合了就停下、维护者当场决定 smoke 还是关）——零新约定，但**一点没省下维护者**，直接违背北极星。
- **塞进 issue body**（正文写一行 `- [ ] 需人工 smoke`）——可行、且不算"新标签"，但不如标签**显式**、机器**稳定可读**；维护者也认同"tag 更显式"。保留为备选，不作主选。

## 后果

- **triage 多一项产出**：判定 smoke-or-not，必要时贴 `needs-smoke`。项目的 triage 标签词汇表（`docs/agents/triage-labels.md`）应补入 `needs-smoke`。
- **部分修订 ADR-0001**：其「零新约定 / 不加标签」不再是绝对铁律——本 ADR 为 `needs-smoke` 开了一个有限、有意识的例外；ADR-0001 的精神（数据主权在标签+正文、不接 GitHub Projects）不变。
- **TB1 只读，所以"自动放行 / 自动关卡"属 Phase 2/3**（写回 GitHub 的能力，ADR-0002）。但 TB1 当下就能用 `needs-smoke` 把「待我验收」这个 **派生阶段** 桶**分两半显示**：带 `needs-smoke` 的＝"等你 smoke"，不带的＝"机器已可判、将来可自动关"——让"盖章"痛点在只读阶段就先变得清楚。
- **工作流可迭代**：本决定是朝北极星的**方向性赌注**，不是定局；真实用起来发现判据/信号不顺，随手感调整（维护者原则：「工作流可以迭代，只要往目标方向优化都可以」）。
- **自动放行的终点＝全自动关卡**（决定第 4 条），同属 Phase 2/3（TB1 只读做不了关卡）；但它给"放行该做多自动"定了方向：过渡档「批量瞄一眼」是为攒信任、降误判风险的临时手段，不是终态。落地时机与"按卡分速"的快慢，随真实手感迭代。
